Status values:

- `resolved_to_neid`: extracted label already exists as its own canonical NEID in the validated MCP view.
- `merged_into_neid`: extracted label was accounted for by entity resolution and maps to a different canonical KG name/NEID.
- `not_resolved`: no canonical NEID was confirmed from the current evidence set.

Columns:

- `extracted_label`: the label in `recordeval_graph_extracted.json`
- `canonical_neid`: the NEID used in the validated MCP interpretation
- `canonical_name_returned`: the name associated with that NEID in the validated view
- `comparison`: how the extracted label compares to the canonical name
